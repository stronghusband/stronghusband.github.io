常见注释

@RestController

作用：这是一个组合注解，相当于@Controller + @ResponseBody
用途：用于标记一个类为 REST 风格的控制器，该类中的所有方法返回的数据（如字符串、对象等）会自动被序列化为 JSON（或 XML 等格式），并直接写入 HTTP 响应体中，而不是返回视图名称（如 JSP 页面）。

@Bean

主要作用是告诉 Spring 容器：“这个方法返回的对象，请把它作为一个 Bean（组件）管理起来，放入 Spring 的 IOC 容器中。
通常，我们使用 @Component、@Service、@Controller 等注解让 Spring 自动扫描并注册类。但是，当你无法修改源代码（例如第三方库的类），或者需要自定义复杂的初始化逻辑时，就需要使用 @Bean

  • | 特性 | @Component (及衍生注解) | @Bean |
    | 所属类 | 直接标注在类上 | 标注在方法上 (通常在配置类中) |
    | 控制力 | Spring 自动实例化，控制较少 | 完全由你控制实例化过程 (可以调用构造函数、设置属性等) |
    |适用对象 | 自己写的代码 (能修改源码) | 第三方库的类、需要复杂配置的对象 |
    | 底层原理 | 类路径扫描 (Classpath Scanning)| 执行配置类中的方法 |
  • . 工作原理详解
    启动时扫描：Spring 容器启动时，会扫描所有带有 @Configuration 的类。
    执行方法：它会执行这些类中所有带有 @Bean 注解的方法。
    注册容器：将这些方法返回的对象实例注册到 Spring IOC 容器中。
    单例管理：默认情况下，@Bean 创建的对象是单例 (Singleton) 的。如果在其他 Bean 中注入该对象，Spring 会直接复用第一次创建的那个实例。

Spring 容器（Spring Container）是 Spring 框架的核心心脏。

简单来说，它是一个超级大工厂，负责创建、配置、组装和管理你应用程序中的所有对象（在 Spring 中称为 Bean）。
如果没有 Spring 容器，你需要自己在代码里写 new User()、new Service()、new Dao()，并且手动把它们互相赋值（比如 service.setDao(dao)）。有了 Spring 容器，这些工作全部由它自动完成，你只需要告诉它“我要什么”，它就会给你准备好。
. 核心功能：它具体管什么？
Spring 容器主要做三件事：

实例化 (Instantiation)：

它负责 new 对象。你不需要写 new 关键字，容器会根据配置帮你创建对象。

依赖注入 (Dependency Injection, DI)：

这是最核心的功能。如果 A 对象需要 B 对象，容器会自动把 B 创建好，然后“塞”进 A 里面（通过构造函数或 Setter 方法）。
例子：UserService 需要 UserDao。你只需在 UserService 里写 @Autowired UserDao dao，容器启动时会自动把 UserDao 的实例赋值给它。

生命周期管理 (Lifecycle Management)：

它管理对象从“出生”到“死亡”的全过程。
包括：创建 -> 初始化（执行 @PostConstruct）-> 使用 -> 销毁（执行 @PreDestroy）。
它还管理对象的作用域（是单例 Singleton，还是每次请求都新建 Prototype）。

. 形象比喻
想象你在经营一家餐厅（你的应用程序）：
没有 Spring 容器时：
你是主厨，每做一个菜（对象），你都要亲自去菜市场买菜（new 对象），洗菜切菜（配置属性），还要自己把盘子和筷子配好（依赖组装）。累且容易出错。
有 Spring 容器时：
Spring 容器就是后勤总管。
你（开发者）只需要写一张菜单（配置文件或 @Configuration 类），上面写着：“我需要一份宫保鸡丁（Bean），里面要加花生（依赖）”。
后勤总管（容器）看到菜单后，自动去采购、制作、摆盘。
当服务员（你的业务代码）喊“我要宫保鸡丁”时（@Autowired），总管直接把做好的菜端上来，服务员只管上菜，不用管菜是怎么来的。

Spring Aop

Spring AOP（Aspect-Oriented Programming，面向切面编程）是 Spring 框架的核心功能之一。
重点：主要目的是将那些与业务无关，却为业务模块所共同调用的逻辑（例如事务处理、日志管理、权限控制等）封装起来，便于减少系统的重复代码，降低模块间的耦合度，
有利于未来的可扩展性和可维护性

重点：代码的重要组成部分（注解、切点、通知、切面）

@Component: 表示当前类被Spring管理
